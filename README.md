# stoerti's Grindware

## Why I do this

I'm stoerti. Hacker since 1989.

Six months ago, I became unemployed in Germany. At 55, with 36 years of experience in offensive security, reverse engineering, and vulnerability research, I'm apparently "overqualified" for the roles I see advertised—or too old to be taken seriously.

I'm not waiting anymore.

This repository is my answer. I'm building grindware—exploitation tools, scanners, and security research code—because I can. Because I've spent decades understanding how systems really work under the hood, and because the industry keeps making the same mistakes it made in the 90s.

I'm offering:

- **Exploits for recent CVEs** – tested, working, documented
- **Scanners** – fast, reliable, built from experience
- **Honeypots and logging tools** – to understand what's actually happening out there
- **A growing collection of offensive security tools**

## What's inside

- **Abyssal** – Telnet vulnerability scanner for CVE-2026-24061, delivering root shells with false-positive detection
- **Doomsday** – IP scanner for large networks and defined ports
- **InstantHoneypot** – Logs connections, payloads, HEX/ASCII, HTTP, and binary data
- **stoertis_toolbox** – A collection of exploitation scripts (Perl/Python)

## Who this is for

- Pentesters who need reliable tools
- Anyone who understands that cybersecurity isn't about certifications—it's about understanding how things break
